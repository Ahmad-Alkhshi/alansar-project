# دليل استخدام نظام المستودع

## نظرة عامة
نظام المستودع يسمح للعمال بتجهيز الطلبات وتسجيل حالة كل مادة (موجودة أو فيها مشكلة).

---

## الخطوات المطلوبة للتفعيل

### 1. تشغيل Migration للـ Database
قبل استخدام النظام، لازم تشغل الـ migration لإضافة الحقول الجديدة:

```bash
# إذا عندك Supabase
# افتح Supabase Dashboard → SQL Editor
# انسخ محتوى الملف: server/migrations/add_warehouse_fields.sql
# شغله

# أو إذا عندك Prisma
npx prisma db push
```

### 2. إعادة تشغيل الـ Server
```bash
cd server
npm run dev
```

### 3. إعادة تشغيل الـ Frontend
```bash
npm run dev
```

## كيفية الاستخدام

### للعمال (صفحة المستودع)

1. **الدخول للصفحة:**
   - الرابط: `https://yoursite.com/warehouse/any-token`
   - يمكن استخدام أي token (مثلاً: `warehouse123`)
   - كل العمال يستخدمون نفس الرابط

2. **اختيار طلب:**
   - يظهر قائمة بكل الطلبات المتاحة
   - اضغط "ابدأ التجهيز" على أي طلب
   - الطلب ينقفل تلقائياً (ما حدا تاني يقدر يشتغل عليه)

3. **تجهيز المواد:**
   - لكل مادة، اضغط:
     - ✓ "موجودة" إذا المادة موجودة وتم تعبئتها
     - ✗ "مشكلة" إذا في نقص أو مشكلة (بيطلب منك تكتب الملاحظة)

4. **إتمام الطلب:**
   - بعد ما تحدد حالة كل المواد
   - اكتب ملاحظات عامة (اختياري)
   - اضغط "تم التجهيز"

5. **إلغاء:**
   - إذا بدك تلغي وتترك الطلب، اضغط "إلغاء"
   - الطلب بيرجع متاح لباقي العمال

---

### للإدارة (صفحة الأدمن)

1. **عرض حالة الطلبات:**
   - كل طلب فيه badge بيوضح حالة التجهيز:
     - ⏸️ "بانتظار التجهيز" (رمادي)
     - ⏳ "قيد التجهيز" (أصفر)
     - ✓ "تم التجهيز" (أخضر)
     - ⚠️ "مشاكل" (أحمر)

2. **فلترة الطلبات:**
   - استخدم القائمة المنسدلة لفلترة الطلبات حسب الحالة
   - مثلاً: اختر "فيها مشاكل" لرؤية الطلبات اللي فيها نقص

3. **تفاصيل المواد:**
   - المواد اللي عليها ✓ تظهر بخلفية خضراء
   - المواد اللي عليها ⚠️ تظهر بخلفية حمراء + الملاحظة

4. **ملاحظات المستودع:**
   - إذا في ملاحظات عامة من العامل، تظهر في صندوق أصفر

## ميزات النظام

### منع التعارض
- لما عامل يفتح طلب، ينقفل تلقائياً
- ما حدا تاني يقدر يشتغل على نفس الطلب
- إذا العامل ما خلص خلال 30 دقيقة، الطلب بيفتح تلقائياً

### التحديث التلقائي
- صفحة المستودع تتحدث كل 5 ثواني
- صفحة الأدمن تتحدث كل 30 ثانية

### تصميم موبايل
- الصفحة مصممة خصيصاً للموبايل
- أزرار كبيرة وواضحة
- سهلة الاستخدام للعمال

---

## API Endpoints الجديدة

```
GET    /api/warehouse/orders              - جلب كل الطلبات
POST   /api/warehouse/orders/:id/lock     - قفل طلب للتجهيز
POST   /api/warehouse/orders/:id/unlock   - إلغاء قفل الطلب
POST   /api/warehouse/items/:id/status    - تحديث حالة مادة
POST   /api/warehouse/orders/:id/complete - إتمام تجهيز الطلب
```

---

## الحقول الجديدة في Database

### جدول `orders`:
- `warehouse_status`: حالة التجهيز (pending, in_progress, completed, has_issues)
- `warehouse_notes`: ملاحظات عامة من العامل
- `warehouse_locked_by`: معرف العامل اللي شغال على الطلب
- `warehouse_locked_at`: وقت قفل الطلب

### جدول `order_items`:
- `warehouse_status`: حالة المادة (pending, checked, issue)
- `warehouse_notes`: ملاحظة على المادة (مثلاً: "ناقص 2 كيلو")

## ملاحظات مهمة

1. **الرابط مشترك:** كل العمال يستخدمون نفس الرابط، ما في login
2. **القفل التلقائي:** الطلب ينقفل لمدة 30 دقيقة، بعدها يفتح تلقائياً
3. **التحديث المباشر:** الصفحات تتحدث تلقائياً، ما في داعي تعمل refresh
4. **الموبايل أولاً:** النظام مصمم للاستخدام على الموبايل

## استكشاف الأخطاء

### الطلب ما عم ينقفل
- تأكد إنه ما في عامل تاني شغال عليه
- انتظر 30 دقيقة وحاول مرة تانية

### البيانات ما عم تظهر
- تأكد إنك شغلت الـ migration
- تأكد إن الـ server شغال
- افتح Console وشوف إذا في errors

### الصفحة ما عم تتحدث
- تأكد من اتصال الإنترنت
- اعمل refresh للصفحة
